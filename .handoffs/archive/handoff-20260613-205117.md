## Decisions & rationale

- **Task = first user prompt (set-once)**: User explicitly requested this. Not git branch (previous default). If no prompt available, git branch is fallback. Only Pi and Claude have pre-prompt hooks; Codex has UserPromptSubmit hook but it fires as part of turn lifecycle.
- **Pi uses terminal capture for last message**: Pi's terminal output format matches ExtractPromptPreview patterns. Simpler than parsing messages array. Other agents don't match so they use hook-based extraction.
- **Claude Stop uses transcript parsing**: Claude Code doesn't put last_assistant_message in the Stop payload directly. We read the JSONL at transcript_path and extract the last assistant message with text content (skipping toolCall-only messages).
- **Codex hooks.json not inline TOML**: Used separate hooks.json for clarity. Also ensured `[features] hooks = true` in config.toml (was `hooks = false`, preventing hooks from running).
- **OpenCode plugin in node_modules/guppi**: OpenCode requires npm-style plugins. File URL registration: `"file:///home/sil/.config/opencode/node_modules/guppi"`. Package.json with `"type": "module"` for ESM.

## Gotchas & landmines

- `go build` without `-a` may use cached packages and not pick up changes to string-template code in agent_setup.go. If generated plugin looks wrong after edits, use `go build -a`.
- Codex hooks are disabled by default (`hooks = false` in `[features]`). agent-setup now sets it to `true`, but existing users who ran old agent-setup have hooks disabled.
- Pi's `agent_start` does NOT have the prompt field. `before_agent_start` does. This was the bug causing Pi to fall back to git branch.
- OpenCode plugin uses Bun's `Bun.spawn()` not Node's `spawnSync`. Plugin runs in OpenCode's Bun runtime.
- The Codex hooks.json format wraps in `{"hooks": {...}}` outer object. This matches the format Codex expects.
- The binary at `/home/sil/guppi/guppi` is NOT committed (it's untracked). The user builds it fresh. Don't commit the binary.

## Mental model

Guppi monitors coding agents in tmux via hooks. Each agent sends events to `guppi notify` via hooks/plugins. The notify command parses stdin JSON to extract: tool name (activity label), user prompt (task, set-once), and agent message (last thing agent said). State is stored in SessionMeta and pushed to the frontend via WebSocket. The sidebar shows the combined display. The two hardest parts were: (1) Claude needing transcript file parsing since it doesn't include last_assistant_message directly, and (2) each agent having its own hook system (Claude hooks.json, Codex hooks.json, Pi TypeScript extension, OpenCode npm plugin).
